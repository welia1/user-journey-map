Open questions.

Each question has a material impact on scope, sequencing, or ongoing ownership. The first is a blocker — the integration cannot be delivered without an answer.

Q · 01Real-time sync· BLOCKER

Does the partner system emit real-time events?

If state changes via another consumer, can the partner push that change to the primary system in real time? Without this, the local mirror drifts and the integration cannot be delivered reliably.

Q · 02Transaction semantics

What shape should the write-back payload take?

When the primary system acts, what payload does the partner expect? API spec, field mappings, and error envelopes need to be locked down before build starts.
